# Настройка

## Что нужно прямо сейчас (только для визарда)

На этом этапе боту не нужны ни база данных, ни LLM, ни облако — **только токен Telegram-бота**.

### Шаг 1. Создай бота и получи токен

1. Открой в Telegram [@BotFather](https://t.me/BotFather).
2. Отправь `/newbot`. Имя — **Deka**, а username придумай свободный (например `deka_kz_bot` или `deka_nalog_bot`) — главное, чтобы не совпадал с гос-ботом @SalyqBot.
3. BotFather пришлёт токен вида `1234567890:AAE...`. Скопируй его.

### Шаг 2. Пропиши токен

```bash
cp .env.example .env
```

Открой `.env` и вставь токен:

```
BOT_TOKEN=1234567890:AAE...
```

### Шаг 3. Установи зависимости и запусти

```bash
npm install
npm run dev
```

Увидишь `✅ Запущен как @твой_бот`. Открывай бота в Telegram, жми **/start** — визард подбора режима работает.

### Проверка кода

```bash
npm test          # юнит-тесты доменной логики (лимиты, подбор режима)
npm run typecheck # проверка типов
```

## Что понадобится позже (этап RAG — пока НЕ нужно)

Оставь эти переменные в `.env` пустыми, дойдём до них на следующем этапе:

- **`GEMINI_API_KEY`** — бесплатный ключ в [Google AI Studio](https://aistudio.google.com/) (генерация ответов + эмбеддинги).
- **`DATABASE_URL`** — бесплатная Postgres-база в [Neon](https://neon.tech/) с расширением `pgvector` (корпус НК и векторный поиск).
- **Cloudflare** — для деплоя в прод (webhook вместо long-polling), тоже free tier.
