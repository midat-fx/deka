# Deka — налоговый ассистент по НК РК-2026

**Живой бот:** [@deka_tax_bot](https://t.me/deka_tax_bot) · **Сайт-калькулятор:** [deka-3st.pages.dev](https://deka-3st.pages.dev)

![CI](https://github.com/midat-fx/deka/actions/workflows/ci.yml/badge.svg)

> **In English (TL;DR):** Deka is a grounded Telegram assistant for Kazakhstan's new 2026 Tax Code. It pairs a deterministic tax-regime wizard (pure, unit-tested logic — no LLM in the legally-sensitive path) with citation-grounded Q&A: BM25 lexical search over the official code (custom Russian stemmer, query expansion, an eval gate at hit@3 = 94%) feeds a Gemini answer layer that must cite an article or is rejected — never a hallucinated tax figure. TypeScript end-to-end, deployed on Cloudflare Workers (webhook) + Pages, D1 telemetry, $0/mo infra. Engineering trade-offs are logged in [DECISIONS.md](DECISIONS.md).

С 1 января 2026 в Казахстане действует новый Налоговый кодекс (№ 214-VIII). Изменилось всё: лимиты самозанятых, упрощёнки, порог НДС; первый полный отчётный цикл — **август 2026** (форма 910 до 15.08). В новых правилах путаются даже бухгалтеры, а официальный бот отвечает заготовками без ссылок на закон. Deka помогает ИП и самозанятым разобраться — и делает это **проверяемо честно**: каждый вывод подкреплён ссылкой на конкретную статью кодекса, а где данных не хватает, Deka говорит «нужно проверить», а не выдумывает.

> ⚠️ Deka — неофициальный сервис, не связан с КГД. Это ориентир, а не налоговая консультация.

## Что уже работает

- **Подбор налогового режима** (бот `/start` + калькулятор на сайте) — детерминированный визард: 3–4 вопроса → рекомендация (самозанятый / упрощёнка / общий режим) с разбором «почему», предупреждениями о порогах и **ссылками на статьи НК РК-2026 на adilet.zan.kz** (Ст. 718, 723, 99). Без LLM — ноль галлюцинаций в юридически значимой части.
- **Поиск по кодексу** — напиши боту вопрос своими словами («когда вставать на учёт по НДС?») и получи **дословные фрагменты нужных статей** со ссылками на первоисточник. Под капотом: BM25 + собственный стеммер русского + расширение запросов («НДС» → «налог на добавленную стоимость», «закрыть ИП» → «прекращение деятельности») + буст заголовков статей и глав. При слабом совпадении — честный отказ, а не догадка.
- **Eval-гейт качества поиска** — golden-set из 18 вопросов с известными правильными статьями, метрики hit@1 / hit@3 / MRR (`npm run eval`). Текущий результат: **hit@3 = 94%, hit@1 = 50%, MRR = 0.71**; eval-driven тюнинг поднял hit@3 с 56% (история итераций — в DECISIONS.md). Гоняется в CI на каждый push — регрессия роняет сборку.
- **Faithfulness-eval LLM-ответов** — отдельный golden-set (`npm run eval:answers`) проверяет, что ответ Gemini *пришёл*, *сослался на верную статью* и *назвал верное число* из кодекса. Текущий результат: **70% faithful**; оставшиеся промахи — не галлюцинации, а честные отказы там, где лексический поиск не поднял нужный фрагмент в топ (цель следующего этапа — векторный поиск).
- **Корпус кодекса** — официальный текст НК (848 статей) распарсен в структурированные карточки «раздел → глава → статья → пункт» с якорями на первоисточник.
- **Телеметрия** — собственная (Telegram ботам статистику не даёт): события в SQLite, приватность by-design (хэш вместо id, тексты не хранятся). `npm run stats` — юзеры, воронка, режимы.
- **Одна логика — все платформы**: сценарий визарда и расчёт лимитов живут в `src/domain/` и переиспользуются ботом (grammY), сайтом (esbuild-бандл) и будущим Mini App.

## Дорожная карта

| Этап | Срок | Итог |
|------|------|------|
| **1. Ядро** | июль | ✅ визард + корпус НК + телеметрия + лендинг + подготовка деплоя |
| **2. RAG + запуск** | июль–август | Ответы на свободные вопросы с цитатой до пункта статьи и честным отказом; golden-set eval с CI-гейтом; деплой 24/7; посев в чатах ИП к волне формы 910 |
| **3. Retention** | август–сентябрь | Напоминания о дедлайнах, трекер оборота с алертами на лимиты, публичный eval-дашборд, кейс на vc.ru/Habr + dev.to |

## Принципы

1. **Grounded или молчит.** Ответ без найденной цитаты не отправляется; при слабом retrieval — честный отказ и отсылка к КГД (1414).
2. **Числа — от МРП, не хардкод.** Все лимиты считаются от одной константы `МРП 2026 = 4 325 ₸` ([src/domain/mrp.ts](src/domain/mrp.ts)); тесты фиксируют каждый порог.
3. **Юридически значимое — детерминировано.** Подбор режима — чистая логика, покрытая тестами; LLM только там, где ответ подкрепляется цитатой.
4. **Качество измеряется.** Этап 2: golden-set + CI-гейт, который блокирует деплой при регрессии качества ответов.

## Стек

**TypeScript** везде · **grammY** (локально long-polling, в проде webhook на **Cloudflare Workers**) · лендинг — статика + **esbuild** на Cloudflare Pages · телеметрия — **node:sqlite** (ноль зависимостей; на Workers → D1) · корпус — **cheerio**-парсер официального HTML · тесты — **Vitest** · этап RAG: **Neon Postgres + pgvector**, **Gemini Flash** (free tier), **promptfoo**. Инфраструктура — $0/мес.

## Запуск локально

```bash
npm install
cp .env.example .env        # впиши BOT_TOKEN от @BotFather
npm run dev                 # бот (long-polling)
npm test                    # 25 юнит-тестов
npm run stats               # статистика использования
npm run build:landing       # собрать сайт в landing/dist
npx tsx scripts/build-corpus.ts   # пересобрать корпус НК из data/raw
```

## Структура

```
src/
  domain/        чистая логика: без сети, без LLM, без Telegram
    mrp.ts         МРП по годам, перевод МРП→тенге
    regimes.ts     режимы, сверенные лимиты, источники (ссылки на статьи НК)
    wizard.ts      детерминированный подбор режима
    flow.ts        сценарий вопросов (общий для бота и сайта)
    format.ts      форматирование сумм
  bot/
    index.ts       точка входа: long-polling (локально)
    wizard-flow.ts рендер сценария в Telegram (inline-кнопки) + телеметрия
  worker.ts      точка входа: webhook на Cloudflare Workers (прод)
  telemetry/     события в SQLite, приватность by-design
landing/         сайт-калькулятор (статика, тот же src/domain)
scripts/         build-corpus (парсер НК), stats
data/corpus/     распарсенный кодекс (relevant.json — 217 статей для ИП)
test/            25 юнит-тестов
eval/            (этап 2) golden-set и eval
```

Инженерные решения и компромиссы — в [DECISIONS.md](DECISIONS.md). Настройка с нуля — в [SETUP.md](SETUP.md).

## Название

**Deka** (от «декларация»). Намеренно не содержит «Salyq», чтобы не пересекаться с официальным гос-ботом [@SalyqBot](https://t.me/SalyqBot) КГД — Deka всегда маркируется как неофициальный независимый помощник.
